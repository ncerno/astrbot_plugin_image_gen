AstrBot × Gemini 3.1 Flash Image 插件技术方案书（初稿 v1）
1. 项目概述
1.1 项目名称
AstrBot 图片生成插件（QQ 场景，基于 Gemini 3.1 Flash Image）

1.2 项目目标
开发一款 AstrBot 插件，使用户可以在 QQ 私聊/群聊中，通过自然语言或专用命令触发：

文生图
图生图
插件使用 AstrBot 已接入的大模型 负责将用户的简短中文描述改写为适合图像模型使用的英文 prompt，再调用 Gemini 3.1 Flash Image 完成图片生成或编辑，并在异步任务完成后自动将结果图片发送回原 QQ 会话。

1.3 设计原则
交互简单，避免与现有其他插件混淆
首版功能聚焦，优先跑通主链路
配置项尽量完整，便于自用和后续调整
保持可扩展，便于未来替换触发词、模型服务和 prompt 策略
节省服务器空间，生成结果发送后自动清理本地临时文件
2. 功能范围
2.1 首版功能范围
文生图
支持以下两种触发方式：

聊天触发：检测固定触发短语，如“帮我画”
指令触发：如 /draw
图生图
支持以下场景：

用户发送一张图片并附带修改描述
指令触发方式，如 /imgedit
Prompt 智能改写
用户输入中文简短描述
插件调用 AstrBot 当前接入的大模型
自动生成适合图像模型使用的英文 prompt
同时生成中文简述，用于回传展示
异步任务处理
图片任务采用异步执行
通过 AstrBot 内建异步任务和定时器轮询状态
完成后自动回发
图片发送与清理
生成成功后，将图片发送到原 QQ 会话
发送后自动删除本地临时文件
兜底清理超时残留文件
2.2 首版不做
多轮连续追图/局部修改链路
图片历史管理与检索
内容安全额外审核层
高级绘图参数（seed、采样器、LoRA 等）
复杂工作流编排
2.3 预留扩展项
修改触发短语
支持更多风格预设
支持更多图像服务 provider
支持回调式任务通知
支持任务队列与更细粒度并发控制
3. 用户交互设计
3.1 聊天触发规则
首版采用强约束触发，避免与其他插件混淆。

文生图触发
命中固定短语时触发，例如：

bot，帮我画一张雨夜中的城市
帮我画一只坐在咖啡馆里的狐狸
推荐规则：

消息中包含“帮我画”
且消息中没有与其他插件冲突的特征
且当前消息不属于明确的非绘图语义
图生图触发
建议单独预留图生图触发短语，例如：

帮我改图
示例：

用户发送图片并说：帮我改图，换成水彩风
用户发送图片并说：帮我改图，把背景换成雪山
建议首版即区分：

文生图：帮我画
图生图：帮我改图
这样比统一都走“帮我画”更稳。

3.2 指令触发规则
建议首版采用以下指令：

/draw <描述>：文生图
/imgedit <描述>：图生图
/drawraw <原始提示词>：跳过 LLM 扩写，直接提交图像模型（可选调试指令）
3.3 异步交互反馈
由于生图是异步任务，建议在提交成功后立即回一条中间消息：

收到，正在生成图片……
收到，正在处理图片编辑任务……
任务完成后再发送：

图片
中文简述
失败时返回：

简要失败原因
是否已自动重试结束
3.4 返回内容
默认返回：

图片
中文简述
可配置返回：

是否附带最终英文 prompt
是否附带参数摘要
默认建议：

不显示最终英文 prompt
配置中可开启
4. 总体技术架构
整体采用模块化设计，分为 7 个模块。

4.1 触发与路由模块
职责
监听群聊/私聊消息
检测聊天触发词
解析命令
判断文生图或图生图
提取文本与图片附件
将请求路由到对应处理流程
输入
AstrBot 消息事件
指令参数
图片附件信息
输出
统一请求对象，例如：

会话信息
用户信息
请求类型（text2image / image2image）
原始描述
原图附件信息
4.2 Prompt Agent 模块
职责
调用 AstrBot 已接入的大模型
将中文描述扩写成英文 prompt
生成中文简述
根据模式注入风格和细节
支持文生图/图生图两种不同的 prompt 模板
处理模式
保守模式：尽量不改变用户意图，只补技术描述
增强模式：默认模式，补全风格、镜头、光影、细节
强创作模式：在用户输入模糊时更主动发挥
输出建议结构

{
  "final_prompt_en": "...",
  "negative_prompt_en": "...",
  "brief_zh": "...",
  "style_tags": ["..."],
  "mode": "text2image"
}
4.3 Gemini Image Provider 模块
职责
封装 Gemini 3.1 Flash Image 的图像生成/编辑能力，屏蔽外部接口差异。

统一接口建议
submit_text_to_image(prompt, options)
submit_image_to_image(image_input, prompt, options)
query_task(task_id)
download_result(result_ref)
设计说明
虽然首版只接 Gemini 3.1 Flash Image，但内部建议保留 provider 抽象，便于以后替换。

4.4 异步任务调度模块
职责
提交图片生成任务
保存 task_id 与上下文映射
使用 AstrBot 内建异步任务/定时器轮询结果
重试失败任务
在任务完成后触发消息回发
任务记录建议字段
task_id
request_type
session_id
platform
user_id
group_id
raw_prompt_zh
final_prompt_en
brief_zh
image_input_ref
status
retry_count
created_at
updated_at
4.5 消息回发模块
职责
根据任务上下文定位原始会话
将生成结果发送到 QQ
携带中文简述
可选附带 prompt
处理发送失败重试
需要确认的官方细节
AstrBot 如何发送图片到 QQ
是否支持直接发送本地文件
是否支持发送 URL
如何回到原始会话上下文
4.6 临时文件管理模块
职责
下载远程图片到本地临时目录
为消息发送提供可用文件路径
发送成功后立即删除
定时清理超时残留文件
设计原则
本地只保留必要的短期文件
不做长期存图
优先节省服务器空间
4.7 配置管理模块
职责
读取插件配置
校验关键配置项
提供统一配置访问接口
支持热更新或重载（如果 AstrBot 支持）
5. 核心流程设计
5.1 文生图流程
用户发送消息或命令
触发与路由模块检测为文生图请求
Prompt Agent 模块将中文需求改写为英文 prompt
调用 Gemini Image Provider 提交任务
记录任务状态并向用户返回“正在生成”
异步任务模块轮询状态
生成成功后下载结果图
消息回发模块发送图片 + 中文简述
删除本地临时文件
更新任务状态为完成
5.2 图生图流程
用户发送图片并附言，或使用 /imgedit
路由模块识别为图生图请求
读取图片附件信息
Prompt Agent 将“修改要求”改写为英文编辑 prompt
Gemini Provider 提交图像编辑任务
异步轮询任务状态
成功后下载编辑后图片
回发图片 + 中文简述
删除临时文件
更新任务状态
5.3 失败处理流程
提交失败
自动重试
超过次数后返回失败原因
状态查询失败
自动重试
记录日志
图片下载失败
再试一次
失败后通知用户任务完成但结果拉取失败
消息发送失败
再试一次
若仍失败，记录日志并保留短期结果以便排查
6. Prompt 策略设计
6.1 输入特点
用户输入通常是简短中文，例如：

“画一只机械猫”
“改成赛博朋克风”
“给我一个可爱的头像”
插件不能直接依赖原始输入作为图像模型 prompt，需要做语义补全。

6.2 Prompt 改写目标
将用户的中文自然语言转为：

英文主 prompt
可选英文负面 prompt
中文简述
风格标签/模式标记
6.3 Prompt 模板建议
文生图模板核心维度
主体
场景
风格
构图
光影
镜头感
材质细节
输出质量要求
图生图模板核心维度
保留原图主体/结构
修改目标
风格迁移要求
光影/氛围变化
不希望改动的部分
6.4 风格系统设计
建议采用双层方案：

风格预设
可选项示例：

auto
realistic
anime
watercolor
cinematic
illustration
自定义风格模板
用户可配置额外风格提示词文本，例如：

“high detail, soft cinematic lighting, clean composition”
注入逻辑
若用户选定风格预设，则注入对应风格片段
若配置了自定义风格模板，则拼接进入 prompt
若均未配置，则交由 LLM 自主补全

9. 异步与定时器策略
9.1 采用方式
基于 AstrBot 内建异步任务和定时器能力实现：

提交任务后立即返回
定时轮询状态
完成后发送结果
9.2 调度策略
建议：

默认轮询间隔：5~10 秒
默认最大轮询次数：按超时预算换算
默认自动重试次数：2 次
9.3 后续可优化方向
若 Gemini/平台支持回调，可升级为回调优先
轮询仅作为兜底
